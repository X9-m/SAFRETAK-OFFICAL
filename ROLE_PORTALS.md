# بوابات سفرتك حسب الدور

## المسارات

- `/` بوابة المسافر الحالية.
- `/office` بوابة المكتب السياحي.
- `/admin` بوابة إدارة المنصة.

تستخدم البوابات الثلاث نفس تدفق التحقق برقم الهاتف والجلسات المخزنة في Supabase، مع التحقق من الدور داخل قاعدة البيانات قبل إعادة أي بيانات أو تنفيذ أي عملية.

## بوابة المكتب

- مؤشرات الخدمات والحجوزات والإيرادات والموظفين.
- إنشاء وتعديل ونشر وإيقاف الخدمات والرحلات.
- تحديث حالة الحجز وحالة الدفع.
- إرسال رسائل مرتبطة بالحجز.
- إضافة وتعديل الموظفين ومستويات الصلاحية.
- عرض سجل المدفوعات والإيرادات.
- تعديل ملف المكتب والصور والموقع والترخيص.

## بوابة الإدارة

- مؤشرات المنصة العامة.
- إنشاء المكاتب وربط رقم دخول لكل مكتب.
- اعتماد وإيقاف المكاتب وتعديل خطة الاشتراك.
- تفعيل وإيقاف المستخدمين.
- تحديث الحجوزات والمدفوعات.
- الإشراف على نشر وتفعيل الخدمات.
- معالجة الشكاوى وطلبات الدعم.
- إدارة الإعلانات والتصنيفات.
- إعدادات العمولة والصيانة وبيانات الدعم.

## عقد قاعدة البيانات

تمت إضافة ربط `travel_offices.phone_owner_id` بحسابات `phone_users`، وتحديث جلسات الهاتف لدعم الأدوار `traveler`, `office`, `admin` مع إبقاء إنشاء الحساب الذاتي محصورًا بالمسافر.

واجهات RPC الجديدة:

- `get_role_portal_snapshot`
- `office_save_service`
- `office_update_booking`
- `office_send_booking_message`
- `office_save_employee`
- `office_update_profile`
- `admin_create_office`
- `admin_update_office`
- `admin_update_user`
- `admin_update_booking`
- `admin_update_service`
- `admin_update_support`
- `admin_resolve_complaint`
- `admin_update_settings`
- `admin_save_ad`
- `admin_save_category`

كل RPC يتحقق من رمز جلسة بطول 64 خانة ومن الدور المطلوب داخل دالة محمية قبل قراءة أو تعديل البيانات. عمليات المكتب مقيدة بالمكتب المرتبط بالحساب ولا يمكنها تعديل بيانات مكتب آخر.

## التحقق المنفذ

- Build وPreview ناجحان على Vercel.
- اختبار OTP لدور المكتب والإدارة داخل Transaction مع Rollback.
- اختبار Snapshot للمكتب والإدارة.
- اختبار إنشاء خدمة وموظف للمكتب داخل Transaction مع Rollback.
- اختبار عزل مكتب عن خدمات مكتب آخر.
- اختبار إنشاء مكتب وإعلان وتصنيف من الإدارة داخل Transaction مع Rollback.
